# Дефекты и их исправления

## Дефект #1: Пустая корзина создаёт счёт с суммой 0

**Описание:** При оформлении заказа с пустой корзиной система создаёт счёт с суммой 0.

**Серьёзность:** Средняя

**Локализация:** `module_c_simple.py`, функция `create_invoice()`

**Причина:** Отсутствует проверка на пустую корзину перед созданием счёта.

**Исправление:** Добавлена проверка `if total == 0: return error`

**Статус:** ✅ Исправлен

**Коммит:** `fix: add validation for empty cart`

---

## Дефект #2: Отсутствие CORS заголовков

**Описание:** Фронтенд не может обращаться к API модулей из-за отсутствия CORS.

**Серьёзность:** Высокая

**Локализация:** Все модули (A, B, C)

**Причина:** Не были добавлены CORS middleware в Flask и FastAPI.

**Исправление:** Добавлен `CORS(app)` в Flask модули и `CORSMiddleware` в FastAPI.

**Статус:** ✅ Исправлен

**Коммит:** `feat: add CORS support to all modules`

---

## Дефект #3: Ошибка при оплате несуществующего счёта

**Описание:** При попытке оплатить несуществующий счёт возвращается 500 ошибка вместо 404.

**Серьёзность:** Средняя

**Локализация:** `module_c_simple.py`, функция `pay_invoice()`

**Причина:** Отсутствует проверка существования счёта.

**Исправление:** Добавлена проверка `if invoice not found: return 404`.

**Статус:** ✅ Исправлен

**Коммит:** `fix: return 404 for non-existent invoice`

---

## Дефект #4: Нет проверки остатков лекарств

**Описание:** Система позволяет заказать лекарство, даже если его нет в наличии.

**Серьёзность:** Высокая

**Локализация:** `module_b_simple.py` и `module_c_simple.py`

**Причина:** Отсутствует интеграция с StockService в основном потоке.

**Исправление:** Добавлен вызов `check_stock()` перед созданием счёта.

**Статус:** ✅ Исправлен

**Коммит:** `feat: add stock validation before invoice creation`

---

## Дефект #5: Retry механизм не работает при Network Error

**Описание:** При недоступности Payment API система падает сразу, без retry.

**Серьёзность:** Высокая

**Локализация:** `logging_and_retry.py`, `RetryPolicy`

**Причина:** Retry политика не была применена к HTTP вызовам.

**Исправление:** Обёрнут вызов API в `@retry` декоратор.

**Статус:** ✅ Исправлен

**Коммит:** `feat: add retry mechanism for payment service`

---

## Дефект #7: Docker контейнеры не подключаются к localhost

**Описание:** Фронтенд в Docker не видит модули через localhost.

**Серьёзность:** Высокая

**Локализация:** `docker-compose.yml`, `frontend_server.py`

**Причина:** В Docker каждый контейнер имеет свой IP.

**Исправление:** Использовать имена сервисов вместо localhost.

**Статус:** ✅ Исправлен

**Коммит:** `fix: use service names for internal Docker communication`
